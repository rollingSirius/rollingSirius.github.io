  • 三体（三册）
  • 1984
  • 失踪的人（卡夫卡）
  • 1Q84（村上春树）

技术类：

  • Python自动化运维
  • 编写高质量代码（改善Python程序的91个建议）
  • Android 设计指南
  • MacTalk人生元编程
  • Openstack 手册

产品类：

  • 人人都是产品经理
  • 结网
  • 从零到一
  • 简约至上
  • 精益创业
  • 写给大家看的设计书
  • 参与感（小米口碑营销内部手册）

其他：

  • 我有一事，生死与之（崔永元）
  • 穷爸爸，富爸爸
  • 性学报告（金赛）
  • 如何高效学习
  • 重病之王（癌症传）

各种记得的，不记得的书加起来也有三十多本了。正式工作以来，不仅不像自己原来想象的那样没有精力和时间去读书，反而读的书比在校期间数量更多，涉及的范围也更广。大概是因为视野大大提高，不再像学生时代，那时对未知事物的了解基本靠想象。

现在已经三月份了，所以这些书应该算是过去八九个月的总和。产品类和技术类的书籍大大增多，因为一方面自己当前的工作就是在做技术，另外自己想要转战产品经理，没有实践的状况下只能靠书来弥补，所以很是看了不少产品相关的书籍。当然，付出也有回报，自己也成功地在一家创业公司找到了产品经理的职位。技术方面也能够感觉到自己一点一滴的进步，从原来对 Python 一无所知到自己用 Django 写个人博客，再到现在已经能够自己搭建一个简易的 Tornado 后台框架，实属不易，其中走了不少弯路。

因为要学的东西实在太多，自己也开始有意或者无意地进行方法论方面的学习。比如说《如何高效学习》以及知乎达人“采铜”的自选集，是快速有效学习方面的技巧，《从零到一》、YC startup class以及《精益创业》都是创业方面的方法论。不过只有方法论还不够，要能够在自己的头脑中能够对一个知识体系迅速建模，不断完善学习路径，并且提高自己将模型迁移到其他知识学习的能力。

自己在十月份的时候做了个“入职四个月总结”，现在回头对照，个人技术、理财以及视野方面都达到了当时的期望。2015 已然过了四分之一，我也在此做下不合时宜的“2015 个人展望”吧。

  • 既然入职了产品经理，能证明自己能力的就是拿出一款合格的产品。第一款不管惊艳与否，平凡与否，最重要的是对得起自己。
  • 技术方面：搬家之后准备入手一台 MacBook Pro，学习下 Swift。
  • 督促自己多写东西，要有一个可以量化的指标。虽然有人说过 “Stop Reading, Start Creating"，但自己功力还远远不够，要一边阅读，一边创作。代码，文字，还有其他什么有趣的东西。

#书单   #总结
